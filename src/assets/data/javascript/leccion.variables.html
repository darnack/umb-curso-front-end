Las variables nos permiten almacenar datos en memoria por medio de referencias nombradas.

JavaScript es un lenguaje de tipado dinámico, eso quiere decir que las variables no están fuertemente tipadas, o no están asociadas a un tipo de dato concreto. Por esto, una variable en JavaScript puede contener un texto, un número, una fecha, un booleano, un objeto, e incluso, una función, sin necesidad de indicar previamente su tipo.

Ámbito de una variable

Las variables tienen diferentes niveles de accesibilidad que son importantes tener en cuenta.

Ámbito Descripción
Global La variable es accesible para toda la aplicación, por lo tanto puede ser leída, reasignada y redefinida desde cualquier línea de código del aplicativo.
Local La variable solamente es accesible dentro de la función donde ha sido declarada, en otro contexto diferente la variable simplemente no existe o no está definida.
Bloque La variable solamente es accesible dentro del bloque donde ha sido declarada, el bloque se define por lo que se encuentra entre corchetes { }.
Módulo La variable es global a nivel del módulo donde es declarada, pero fuera del módulo no existe.

Declarar una variable

Existen 4 formas de declarar una variable en JavaScript.

💡El ámbito de las variables declaradas dependerá de dónde se declaren.

Usando var

La palabra clave var permite declarar una variable que puede ser redefinida y reasignada.

Usando let

La palabra clave let permite declarar una variable que puede ser reasignada, pero no redefinida.

Usando const

La palabra clave const permite declarar una constante, esta no puede ser reasignada ni redefinida.

De forma implícita

Si se omite la palabra clave antes de la declaración, es decir solo escribiendo el nombre de la variable, también es posible realizar una declaración. Esta se comportará como si si utilizara var.

💡 Las variables const y las variables declaradas implícitamente deben obligatoriamente ser inicializadas, es decir, darles un valor al ser declaradas. Mientras que con var y let se puede omitir el valor al momento de la declaración.

💡 Las variables no inicializadas tendrán por defecto el valor undefined.
