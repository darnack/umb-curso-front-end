El manejo de eventos es una de las características más importantes en JavaScript, ya que el objetivo fundamental de este lenguaje es agregar interactividad a las aplicaciones web. Esto se consigue principalmente con la implementación de los eventos.

Tipos de evento

La interactividad de una aplicación depende de qué dispositivos periféricos maneje, para móviles los eventos son más especializados porque disponen de una pantalla táctil, para dispositivos de escritorio o portátiles los eventos se centran en el ratón y el teclado.

Eventos del ratón

El ratón o mouse es un periférico con grandes posibilidades de interacción, JavaScript permite identificar eventos del ratón como lo son:

Evento Descripción
click Se produce al pulsar el botón izquierdo del ratón y posteriormente soltarlo.
focus Se produce cuando se da clic sobre un elemento produciendo un foco sobre este.
dragdrop Se produce cuando un usuario suelta algo que había arrastrado sobre la página web.
blur Se produce cuando se da clic fuera de un elemento con foco, haciendo que se pierda el foco sobre este.
mousedown Se produce en el momento de pulsar cualquier botón del mouse.
mousemove Se produce cuando el ratón se mueve por la pantalla.
mouseout Se produce cuando el puntero del ratón sale del área ocupada por un elemento.
mouseover Se produce cuando el puntero del ratón entra en el área ocupada por un elemento.
mouseup Se produce en el momento que el usuario suelta el botón del ratón, que previamente había pulsado.
mouseup Se produce en el momento que el usuario suelta el botón del ratón, que previamente había pulsado.

Eventos del teclado

Los eventos del teclado son especialmente útiles en dispositivos de escritorio y portátiles para crear atajos, y acciones en los video juegos.

Evento Descripción
keydown Se produce en el instante que un usuario presiona una tecla.
keypress Se produce cuando el usuario deja pulsada una tecla un tiempo determinado. Antes de este evento se produce un keydown.
keyup Se produce cuando el usuario deja de apretar una tecla. Se produce en el momento que se libera la tecla.

Eventos táctiles

En el caso de dispositivos de pantalla táctil JavaScript permite manejar de manera primitiva los eventos de toque sobre las pantallas.

Evento Descripción
touchstart Este se genera al hacer cualquier toque a la pantalla, sin importar su duración o movimientos realizados.
touchend Este se ejecuta una vez se deja de tocar la pantalla o el objeto que tiene asignado el manejador de eventos.
touchmove Este es ejecutado una vez se desliza o desplaza el dedo del usuario, por encima de la pantalla u objeto que está siendo controlado a través del manejador de eventos.

Eventos del formulario

Estos se producen sobre elementos de captura de datos en los formularios.

Evento Descripción
change Se desata este evento cuando cambia el estado de un elemento de formulario.
reset Se desata en el momento que un usuario hace clic en el botón de reset de un formulario.
select Se ejecuta cuando un usuario realiza una selección de un elemento de un formulario.
submit Ocurre cuando el usuario presiona el botón de enviar el formulario. Se ejecuta justo antes del envío.

Eventos del navegador

Estos eventos están relacionados a cambios sobre el navegador o eventos de carga.

Evento Descripción
abort Se produce cuando un usuario detiene la carga de una imagen, al detener la carga de la página o cerrar la página.
error Se produce cuando no se puede cargar un documento o una imagen y esta queda rota.
load Este evento se desata cuando la página ha terminado de cargarse.
resize Se produce cuando se redimensiona la ventana del navegador.
unload Se produce al abandonar la página.
scroll Este evento se produce cuando se realiza scroll o desplazamiento.
DOMContentLoaded Ese produce al finalizar la carga del DOM, se aplica al objeto document.

Manejadores de eventos

Hay varias manejas de manejar eventos en JavaScript.

Por atributo

Las etiquetas de los elementos HTML tienen atributos de eventos soportados nativamente. Para implementarlos solo basta con escribir código JavaScript directamente asociado a estos atributos.

Aunque es muy recomendable invocar solamente funciones JavScript, en lugar de escribir código directamente sobre estos atributos, de esta manera se mantienen un código más legible.


Por propiedad

Una manera más ordenada de manejar los eventos es utilizando las propiedades de evento de los elementos HTML. Para ello primero se debe seleccionar el elemento del DOM, posteriormente se debe asignar el evento a la propiedad correspondiente.

Agregando una escucha

La forma más sofisticada de implementar manejadores de eventos es agregándolos por medio de la función addEventListener. Este permite agregar eventos de forma dinámica a cualquier elemento HTML.

💡 El nombre del evento a diferencia de los dos casos anteriores, no es precedido por la palabra on, así, en lugar de introducir onclick, solo debe escribirse click.

Interfaz de evento

La interfaz de evento Event es un objeto que contiene propiedades pertinentes al evento, permite conocer información relevante como el elemento que originó el evento o las coordenadas desde dónde se originó.

Para acceder a la interfaz Event se debe pasar como primer parámetro de entrada a la función asignada al manejador del evento.

La interfaz Event tiene muchas propiedades, algunas de ellas son:

Propiedad Descripción
altKey Indica si la tecla <alt> fue pulsada durante el evento.
button Devuelve el botón del ratón.
charCode Devuelve el valor Unicode de una tecla de carácter que fue apretada como parte de un evento keypress.
clientX Devuelve la posición horizontal del evento.
clientY Devuelve la posición vertical del evento.
ctrlKey Indica si la tecla <ctrl> fue apretada durante el evento.
currentTarget Devuelve una referencia al objetivo actual registrado para el evento.
isChar Indica si el evento produce o no una tecla de carácter.
keyCode Devuelve el valor Unicode de la tecla presionada.
screenX Devuelve la coordenada horizontal del evento en la pantalla.
screenY Devuelve la coordenada vertical del evento en la pantalla.
shiftKey Indica si la tecla <shift> fue presionada cuando el evento fue disparado.
target Devuelve la referencia del objeto que disparó el evento.
timeStamp Devuelve el momento de creación del evento.
type Devuelve el nombre del evento (distingue mayúsculas y minúsculas).
which Devuelve el valor Unicode de la tecla en un evento del teclado, sin importar el tipo de tecla que se presionó.

También cuenta con algunos métodos como lo son:

Método Descripción
preventDefault Cancela el comportamiento nativo del evento.
stopPropagation Para la propagación de los eventos más allá en el DOM.
