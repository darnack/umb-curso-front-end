Un objeto es una estructura de datos que tiene propiedades, las cuales a su vez pueden hacer referencia a un dato, una variable, un arreglo, una función, u otro objeto. Los objetos son estructuras de datos jerarquizadas.

El manejo de objetos es fundamental para el tratamiento de datos, enviar y recibir información empaquetada dentro de objetos estableciendo comunicación con servidores y servicios remotos, manejar estructuras de datos relacionadas a la parte visual del aplicativo web y muchos otros propósitos.

💡 Existen dos maneras de declarar un objeto, por medio del constructor o por medio de notación JSON.

Constructor

Por medio del constructor Object y del operador new es posible declarar un objeto. Las propiedades del objeto son declaradas de forma dinámica simplemente al nombrarlas y asignarles un valor.

Notación JSON

Las siglas JSON significan JavaScript Object Notation o Notación de Objetos de JavaScript. Ha surgido como una alternativa a la notación XML y ha tenido un éxito y acogida definitiva en la industria del software, convirtiéndose en un estándar para la representación de objetos.

💡 La notación JSON hace uso de 4 simples conceptos.

Objeto

Un objeto JSON es representado por los corchetes { }. Unos corchetes vacíos representan un objeto vacío.

Llave / Valor

Las propiedades de un objeto JSON son representadas en forma llave/valor utilizando el caracter clave : como separador entre la llave y el valor, y la coma , como separador entre las diferentes propiedades del mismo objeto.

Colecciones

Las colecciones en un objeto JSON son representadas por llaves [ ]. Cada elemento de la colección puede ser un dato primitivo o un objeto JSON

Anidación

Las propiedades de un objeto JSON pueden hacer referencia a otro objeto JSON anidado, esta anidación puede repetirse indefinidamente tanto como sea necesario para describir un objeto con el nivel de detalle requerido.

Acceso a las propiedades

Para acceder a las propiedades de un objeto y leerlas o modificarlas, generalmente se utiliza el operador punto ., aunque también es posible acceder a estas como si se trataran de elementos de un arreglo.
